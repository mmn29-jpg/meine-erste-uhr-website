Der Git-Workflow

Vom Hochladen im Browser zum echten Arbeiten mit Git auf deinem Computer.

Die Idee dahinter

  • Dein Projekt liegt zweimal: einmal auf deinem Computer und einmal auf GitHub.
  • Du arbeitest auf deinem Computer und schickst deine Änderungen zu GitHub.
  • So musst du keine Dateien mehr im Browser hochladen.
  • Git merkt sich jede Version mit einer eigenen Nachricht von dir.

1 Projekt auf den Computer holen

Hole dein bestehendes GitHub-Projekt einmalig auf deinen Computer.

git clone https://github.com/mmn29-jpg/meine-erste-uhr-website.git
cd meine-erste-uhr-website

2 Der Arbeits-Kreislauf

Diese drei Befehle nutzt du immer wieder, nachdem du eine Datei geändert hast:

git add index.html              # Änderung auswählen
git commit -m "Titel geändert"  # Version speichern
git push                        # zu GitHub schicken
Bearbeiten → auswählen → beschreiben → hochladen

3 Überblick behalten

Diese zwei Befehle ändern nichts. Sie zeigen dir nur, was los ist:

git status   # Was habe ich geändert?
git log      # Welche Versionen gibt es schon?

Wichtig zu wissen

Schreib bei git commit immer eine kurze, klare Nachricht, was du geändert hast. Das ist viel besser als die automatische Nachricht „Add files via upload" aus dem Browser.
